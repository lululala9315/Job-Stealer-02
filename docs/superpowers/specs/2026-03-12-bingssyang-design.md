# 감정 기반 맛집 큐레이션 서비스 설계

## Context

결정 장애가 있거나 뻔한 위로보다 팩폭을 원하는 사용자를 위한 맛집 추천 서비스. 겉으로는 정중한 존댓말이지만 내용은 냉소적으로 비꼬는 '큐레이션' 컨셉. 내부 공유용 캐주얼 프로젝트.

## 아키텍처

**풀 SPA** — Vite + React + Tailwind CSS, 백엔드 없음, GitHub Pages 배포

### 기술 스택
- **프론트엔드**: Vite + React 19 + Tailwind CSS
- **감정 분류**: Gemini API (무료 티어, 분당 15회)
- **맛집 검색**: 네이버 지역 검색 API (리뷰수/평점 제공)
- **배포**: GitHub Pages (private repo 권장)
- **API 키 관리**: 환경변수(.env)로 빌드 타임 주입, .gitignore에 .env 추가. 내부 공유용이므로 클라이언트 호출 허용하되 API 사용량/도메인 제한 설정 권장

## 핵심 플로우 (2단계)

### Step 1: 입력 화면
- 중앙 정렬, 여백 풍부한 매거진 스타일
- "오늘 기분이 어떠세요?" 질문 + textarea 입력
- 빈 텍스트 제출 방지 (유효성 검사)
- 보내기 버튼

### Step 2: 결과 화면 (멘트 + 맛집)
- **상단**: 감정별 고정 이미지 + 멘트 (한 글자씩 타이핑 효과, ~50ms/글자)
- **하단**: 맛집 카드 5개 (식당명, 리뷰수, 평점, 거리, 네이버 플레이스 링크)
- **최하단**: "다시 해보기" 버튼
- Step 1 → Step 2 전환 시 fade 애니메이션

## 감정 분류 및 멘트 로직

### 감정 분류
- 사용자 텍스트 → Gemini API → 5가지 감정 중 하나로 분류
- 감정 매핑 테이블:

| 한글 | 영문 키 | 설명 |
|------|---------|------|
| 분노 | anger | 화남, 짜증, 열받음 |
| 우울 | depression | 슬픔, 우울, 외로움 |
| 번아웃 | burnout | 지침, 과로, 탈진 |
| 기쁨 | happy | 기쁨, 설렘, 신남 |
| 무기력 | lethargy | 불안, 무기력, 의욕 없음 |

### Gemini 프롬프트 설계
```
시스템: 사용자의 텍스트를 읽고 감정을 분류하세요.
반드시 다음 5개 중 하나만 답하세요: anger, depression, burnout, happy, lethargy
다른 설명 없이 감정 키워드 하나만 응답하세요.

사용자: {userInput}
```
- 응답이 5개 키워드에 없을 경우 → fallback으로 `lethargy` 사용

### 멘트
- 감정당 10개 이상의 미리 작성된 멘트 (사용자 제공 예정)
- 분류된 감정에 매칭되는 멘트를 랜덤 선택
- 한 글자씩 타이핑 효과로 노출 (~50ms/글자)

### 감정별 이미지
- 감정당 1장, 고정 이미지
- MVP 단계에서는 Unsplash 등 무료 이미지 사용, 추후 교체

## 맛집 검색 (네이버 지역 검색 API)

### 위치
- 기본값: 문정역 (37.485, 127.122)
- 옵션: 사용자 현재 위치 (Geolocation API)
- 위치 권한 거부 시: 문정역 기본값 사용 (별도 안내 없이 자연스럽게 처리)

### 검색 로직
1. 감정별 메뉴 키워드로 네이버 지역 검색 (사용자 제공 예정)
2. 키워드는 배열로 관리하여 다양한 결과 확보
3. 검색 결과를 리뷰 수 기준 내림차순 정렬
4. 상위 5곳 추출

### 출력 항목
- 식당명
- 리뷰 수
- 평점
- 거리
- 네이버 플레이스 링크

## 에러 처리

| 상황 | 처리 |
|------|------|
| Gemini API 실패/제한 초과 | "잠시 후 다시 시도해주세요" 안내 |
| 네이버 API 검색 결과 0건 | "근처에 맛집이 없나 봐요. 키워드를 바꿔볼까요?" |
| 네트워크 오류 | "인터넷 연결을 확인해주세요" |
| 빈 텍스트/의미 없는 입력 | 보내기 버튼 비활성화 (최소 2글자) |

## UI/UX 가이드

### 톤 & 매너
- 극도의 미니멀리즘, 여백 풍부
- 매거진 스타일 타이포그래피 강조
- 모바일 퍼스트 반응형 (내부 공유 시 모바일 접근 가능성 높음)

### 타이포그래피
- 제목: Serif(명조 계열) — Noto Serif KR
- 본문: Sans-serif(고딕 계열) — Pretendard

### 인터랙션
- 멘트: 한 글자씩 타이핑 효과 (~50ms/글자)
- 화면 전환: fade 애니메이션

### 색상
- 미니멀 흑백 톤 기반 (상세 디자인은 추후 확정)

## 사용자 제공 필요 항목
1. **감정당 10개 이상 멘트** (5가지 감정 × 10개 = 최소 50개)
2. **감정별 추천 메뉴 키워드** (5가지, 감정당 2~3개)
3. **Gemini API 키** (Google AI Studio에서 발급)
4. **네이버 API 키** (Naver Developers에서 Client ID + Client Secret 발급)

## 검증 방법
1. `npm run dev`로 로컬 서버 실행
2. 텍스트 입력 → Gemini API 감정 분류 확인
3. 분류된 감정에 맞는 멘트 + 이미지 노출 확인
4. 네이버 API 맛집 검색 결과 5개 정상 노출 확인
5. 타이핑 효과, fade 전환 등 인터랙션 확인
6. "다시 해보기" 버튼으로 초기화 확인
7. 에러 케이스 테스트 (빈 입력, API 실패 등)
8. 모바일 반응형 확인
9. `npm run build` → GitHub Pages 배포 테스트
